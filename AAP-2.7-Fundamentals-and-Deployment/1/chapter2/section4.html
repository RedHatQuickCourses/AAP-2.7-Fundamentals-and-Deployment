Operator Deployment LAB

Please be aware that setting up a Operator Lab will take about 15-20 minutes. So while the deployment is going on please wait for the pods to come up before accessing the Ansible Automation Platform. You can proceed with the installation procedures:

  1. Log in to the Red Hat OpenShift Container Platform using your credentials form the RHDP login page.

  2. Navigate to Ecosystem Software Catalog and search for Ansible Automation Platform.

  3. Search for Ansible Automation Platform and click Install.

  4. Select a stable-2.7.

  5. Select namespace as default which will be aap and wait for the Operator to be installed and shows the status as successful.

  6. Click Install.

  7. Go to Installed Operators and there you will see Ansible Automation Platform operator installed click on it.

  8. Select project as aap if it’s not aap.

  9. There scroll to the option Ansible Automation Platform and click on Create instance which will deploy all the components of AAP.

  10. Provide the Name on which will be used to create the resources for that AAP like network routes,secrets.

    There are many other options that we can customize according to the requirement but in this we will be taking care of very basic deployment.

  11. Click on Create it will deploy the resources which will take around 20 minutes.

  12. On openshift dashboard Click on Networking and then Routes and click on the ansible route or the Name given in Step 11 above.

  13. On openshift dashboard go to Workloads and then Secrets and search for admin . click on it and copy the password.

  14. Provide the Red Hat login ID to get the subscription for attaching it to the AAP and 60-day subscription to Red Hat Ansible® Automation Platform or Red Hat Developer Subscription for Individuals subscription can be used for this lab.

  15. Submit it and run the Project sync of the Demo project to confirm the job is getting executed.

Congratulations! You have successfully deployed the Ansible Automation Platform using the Operator method. You can now access the AAP web interface using the admin user and the password obtained from the secrets in OpenShift.
